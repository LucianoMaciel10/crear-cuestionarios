This is a real software project, not a prototype.

Never generate simplified implementations.

Always preserve:

- existing architecture
- folder structure
- naming conventions
- separation of responsibilities
- existing design decisions

Before modifying code:

inspect the current implementation.

Extend the existing architecture whenever possible.

Never duplicate services.

Never duplicate business logic.

Reuse existing services before creating new ones.
